Instrucciones para el Proyecto Final 1 - Currículum Vitae en Línea
Especificaciones del Proyecto

    1- Crea un archivo HTML llamado index.html.

    2- Utiliza la declaración <!DOCTYPE html> al principio del archivo para indicar que es un documento HTML.

    3- Crea una sección head que incluya un elemento title con el texto "Mi Currículum Vitae".

    4- Dentro de la sección body, escribe código HTML para crear un currículum vitae en línea que cubra los siguientes aspectos:

        Resumen o declaración de objetivos

        Educación (lista de títulos, escuelas y fechas)

        Experiencia laboral (lista de títulos de trabajo, empleadores, fechas y responsabilidades)

        Habilidades (lista de habilidades relevantes o áreas de experiencia)

        Premios, certificaciones u otros logros (lista de premios relevantes, certificaciones u otros logros)

    5- Utiliza etiquetas HTML apropiadas para dar formato al contenido de cada sección, como h1 para encabezados de sección y ul u ol para listas.

    6- Utiliza etiquetas HTML para agregar una foto de perfil tuya.

    7- Utiliza etiquetas de anclaje relevantes para crear un sitio web de varias páginas, enumerando otros aspectos de tu currículum vitae, como Pasatiempos y Detalles de contacto.

    8- Agrega un elemento footer con tu nombre y cualquier información de derechos de autor u otras renuncias. (Sugerencia: utiliza la documentación de MDN para cosas que no sepas cómo hacer: https://developer.mozilla.org/en-US/docs/Web/HTML/Element/footer

    9-Guarda el archivo index.html y ábrelo en un navegador web para asegurarte de que se muestre correctamente.

    10- Agrega tu sitio web a tu GitHub para comenzar a construir tu portafolio.

    11- Publica tu sitio web utilizando las páginas de GitHub y compártelo aquí (en la sección de Preguntas y Respuestas) con otros estudiantes.

    12- Comenta y haz sugerencias sobre los proyectos de otros estudiantes.

Requisitos

    Para mantener el proyecto dentro del alcance (tiempo, energía, recursos), solo puedes utilizar HTML para completar este proyecto. Incluso si conoces CSS y JS (lo cual asumimos que no conoces todavía), se espera que escribas el sitio web solo utilizando HTML.

    Imagina que eres un desarrollador profesional trabajando en este proyecto. Habrá cosas que no sepas, cosas que quieras hacer pero aún no hayas aprendido cómo hacerlas. Como desarrollador profesional, necesitas conocer los límites de tus habilidades y buscar otras cosas en Google, la documentación de MDN, internet.

Preguntas frecuentes

1. ¿Puedo usar estos proyectos finales en mis entrevistas de trabajo?

¡Sí! Ese es el punto de estos proyectos finales. A diferencia de los proyectos del curso, que tienen código de solución y recorridos en video, los Proyectos Finales están destinados a ser completamente tu "propio trabajo". Hay más de un millón de estudiantes que se han inscrito en este curso de desarrollo web en todo el mundo. Si agregas un proyecto del curso a tu portafolio, es muy probable que tu cliente sepa que es de este curso. Los proyectos finales, por otro lado, no tienen código de solución, ni recorridos, ni especificaciones de diseño. Es puramente tu creación, por lo que definitivamente se puede considerar como tu propio trabajo y agregarlo a tu portafolio.

Cuando contrato desarrolladores, siempre pido un portafolio. Considero que lo que han construido por sí mismos es mucho más importante que a qué universidad fueron o si tienen un título en informática. Muchos de mis amigos que son CEOs en multinacionales y startups están de acuerdo.


2. ¿Qué pasa si no recuerdo cómo hacer algo, aunque haya visto todos los videos?

Ver un video a menudo no es suficiente para aprender una habilidad. Por eso, estos proyectos finales son una gran oportunidad para identificar tus puntos débiles. Mientras construyes, te encontrarás atascado o olvidarás cómo hacer las cosas. ¡Genial! Es entonces el momento de revisar lecciones anteriores o documentación en línea para llenar tus lagunas de conocimiento.
